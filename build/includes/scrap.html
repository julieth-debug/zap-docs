Basic penetration test

All requests that are proxied through ZAP or initialised by tools like the spider are passively scanned: (In progress, This hyper link will be linked to the auto generated API catalogue). Passive scanning is always performed because it is completely safe - ZAP just looks at the requests and responses rather than making any additional requests. This is good for finding problems like missing security headers or missing anti CSRF tokens but is no good for finding vulnerabilities like XSS which require malicious requests to be sent - thats the job of the active scanner.
